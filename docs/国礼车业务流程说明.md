# 国礼车业务流程说明

## 一、整体状态流转图

```
方案中 (draft)
    │
    │ 提交审核 [POST /api/workflow/:id/submit]
    ▼
待审核 (pending_review)
    │
    │ 品牌审核通过 [POST /api/workflow/:id/approve]
    ▼
待审核 - 外事审核
    │
    │ 外事审核通过 [POST /api/workflow/:id/approve]
    ▼
待审核 - 工艺审核
    │
    │ 工艺审核通过 (需检查工艺清单完整性)
    ▼
制作中 (in_production)
    │
    │ 标记待交付 [POST /api/workflow/:id/ready-delivery]
    │ (需检查所有工艺完成当前版本)
    ▼
待交付 (pending_delivery)
    │
    │ 确认交付 [POST /api/workflow/:id/deliver]
    ▼
已交付 (delivered)
    │
    │ 归档 [POST /api/workflow/:id/archive]
    ▼
归档 (archived)
```

### 4. 驳回流程

**代码位置**：[workflow.js](file:///Users/huangding/Documents/SOLOCODE%203/0619/mbp/zj-00393-giftdove-5/src/routes/workflow.js#L121-L175)

**状态流转**：`待审核` → `方案中`

**驳回原因记录位置**：
- **表**：`audit_logs`
- **关键字段**：
  - `action`: `reject`
  - `reason`: **驳回原因（必填）**
  - `review_level`: 驳回的审核级别
  - `reviewer`: 驳回人
  - `design_version`: 驳回时的设计版本

**驳回后自动操作**：
1. 设计版本号 `design_version` +1（版本升级）
2. 所有工艺完成状态重置：[resetCraftsForRework](file:///Users/huangding/Documents/SOLOCODE%203/0619/mbp/zj-00393-giftdove-5/src/models/craftItem.js#L122-L132)
3. 新增一条 `rework` 动作的审核日志，记录版本升级和工艺重置说明
4. `national_gifts.status` 回到 `方案中`
5. `production_start_date` 和 `production_end_date` 清空

**关键代码**：
```javascript
// 驳回时的处理在 [workflow.js#L143-L150]
addAuditLog(
  req.params.id,
  currentLevel,
  "reject",   // 驳回动作
  reviewer,
  reason,     // 驳回原因
  oldVersion,
);
```
